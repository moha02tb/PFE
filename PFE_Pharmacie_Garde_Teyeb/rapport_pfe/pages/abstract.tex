This final-year project presents the design and implementation of \textbf{PharmacieConnect}, a web and
mobile platform for pharmacy search, on-duty pharmacy schedule consultation, and pharmaceutical data
administration. The project was developed around a practical need: on-duty pharmacy information is
often scattered, difficult to update, and not always easy to use on a mobile device.

The proposed solution is built around three parts. The backend, developed with \textbf{FastAPI}, centralizes
data, exposes REST APIs, and applies the main business rules. It also secures access through JWT
authentication. The administration dashboard, built with \textbf{React} and \textbf{Vite}, allows authorized
users to manage pharmacies, duty schedules, medicines, CSV imports, analytics, and audit logs. The
mobile application, developed with \textbf{Expo} and \textbf{React Native}, allows citizens to search for
pharmacies, consult on-duty schedules by date, use geolocation, and access the medicine catalog.

This work also covers several software engineering aspects, including data modeling with SQLAlchemy,
input validation, role-based access control, audit logging, backend testing, and a clear separation
between client interfaces, business services, and data persistence.

\textbf{Keywords:} on-duty pharmacy, FastAPI, React, Vite, Expo, React Native, JWT, REST API,
SQLAlchemy, dashboard, mobile application.
